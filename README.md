# PoE-Base-Price-Output

Public mirror of [PoE-Base-Pricer](https://github.com/T1mothy-L/PoE-Base-Pricer)'s `latest.json`, with auto-regenerated PoE2 item filters that fold the latest white-base prices into visual tiers.

## Files

- **`latest.json`** — slim `[{base, min_ilvl, median_exalts}, ...]`, mirrored from the private price-tracker repo by the `Publish Output` workflow.
- **`*.filter`** (source) — base filter files you want price-tier rules prepended to. Drop any `.filter` here, commit, and the workflow will produce a derived twin.
- **`*updated.filter`** (derived, auto-committed) — what you subscribe to in-game. Generated by `update_filter.py` on every push.
- **`update_filter.py`** — the generator. Stdlib only. `--headless` mode is what the workflow uses; double-clicking on Windows (or `python update_filter.py`) opens a small Tk UI for local tier tweaking.

## How it works

```
PoE-Base-Pricer (private)                    this repo
  laptop runs poe2_price_tracker.py
  commits prices.db + latest.json
        │
        ▼
  Publish Output workflow ─── PAT push ────► latest.json updated
                                                  │
                                                  ▼
                                        Regenerate filters workflow
                                                  │
                                                  ▼
                                        *updated.filter regenerated,
                                        committed via GITHUB_TOKEN
                                        (no recursion: token-pushed
                                        commits don't fire workflows)
```

Re-runs are idempotent: the managed block between `# BEGIN AUTO` and `# END AUTO` is replaced in place, so a derived file is never duplicated and the rest of its content stays untouched. Source filters are **never** modified by the workflow — only their derived twin is written.

## Naming convention

For each source `Foo.filter`, output is written to `Foo updated.filter` in the same directory. Files whose stem already ends in ` updated` are skipped during source discovery, so you can't end up with `Foo updated updated.filter`.

## Adding / removing filters

- **Add**: drop a new `<name>.filter` at the repo root and push. The workflow generates `<name> updated.filter` on the next run.
- **Remove**: delete the source `<name>.filter` and push. The matching `<name> updated.filter` is left in the repo as an orphan (the workflow only writes for currently-present sources); delete it manually if you want.

## Tier ladder

`update_filter.py` buckets each priced base into S / A / B by median exalt value. Defaults at the top of the script:

| Tier | Floor (ex) | Visual                          |
|------|-----------:|---------------------------------|
| S    | 150        | font 45, white-on-red bg, Red Star icon, alert 6 |
| A    | 50         | font 45, red-on-white bg, Red Circle icon, alert 3 |
| B    | 10         | font 40, white-on-dark-red, Brown Circle, alert 2 |

Items below B's floor get no rule (fall through to whatever the rest of the source filter does). Items with no listings in `latest.json` get a `Show` rule with a black bg / white text / alert 2 — visible but understated.

Change `DEFAULT_THRESHOLDS` in `update_filter.py` and commit if you want to tune.

## In-game

Drop the relevant `<name> updated.filter` into `%userprofile%\Documents\My Games\Path of Exile 2\` (or use the auto-subscribe mechanism for the official filter repository if you've enabled it). Reload in-game via Escape → Options → UI → Item Filter.
